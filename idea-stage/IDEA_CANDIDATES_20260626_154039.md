## Mechanism Diagnosis

pair-level false-negative correction 反复不转化为节点分类 accuracy，核心原因可能不是“pair 还没选准”，而是优化对象层级错了：

1. **label agreement 不是 linear boundary preservation**：同类 pair 被拉近可能提升局部 agreement，但也可能压缩子类、少数类或边界节点的局部 margin。
2. **low-overlap 不等于有用信息**：低 overlap 证明不是 kNN/PPR/CAST 复制品，却不能证明这些 pair 携带分类所需的新判别方向。
3. **relation smoothing 容易过度吞并 granularity**：pair edit 会把节点间关系离散化为 positive/neutral/negative，但节点分类需要保留类内多簇和类间近邻边界。
4. **offline diagnostics 与 downstream evaluator 不同目标**：label agreement、margin-rank、uniformity delta 都是表示几何代理，Logistic Regression accuracy 取决于 train/val 可学到的全局线性分割。
5. **false-negative 修正只处理 denominator 局部项**：如果 representation 已经在 augmentation/message passing 中丢失边界频带或有效秩，后续 pair 修补难以恢复。

因此下一轮应从 objective-level / representation-level 重新定义任务：让预训练目标直接保护分类边界、有效秩、局部高频残差信息和合适的 alignment-uniformity tradeoff，而不是继续寻找更“像真阳性”的跨节点 pair。

## Top1 Choice

选择 **SPECTRA-GCL**。

理由：

- 它最直接吸收 CAST/BEACON 的共同失败：不再追求 pair diagnostics 更好，而是直接保护 representation-level 的 boundary energy、rank 和 redundancy。
- 它能把 false-negative 问题改写成更强的机制假设：pair-level repulsion/attraction 在同配图上会误伤，但真正影响分类的是表示是否保留可线性分割的局部边界与有效维度。
- 它的最小 smoke 成本低，且失败可解释：如果 negative-free + boundary-energy guard 仍低于 GRACE/CCA/Barlow control，就说明当前数据集上 false-negative 修正不是主要瓶颈。
- 它不使用 test label 选阈值；所有权重和 early stop 仍走 train/val 或固定 smoke 配置。

## Kill Rules for Top1

- SPECTRA full 不超过 negative-free baseline，且 boundary-energy/effective-rank diagnostic 没有改善：`KILL_MECHANISM`。
- diagnostic 改善但 accuracy 不改善，重复 BEACON 模式：`REVISE_OBJECTIVE_NOT_DIAGNOSTIC`，不得继续调权重。
- 任何提升只来自更大 projection/head 或训练预算：`INVALID_FAIRNESS`。
- random-spectrum guard 与 SPECTRA full 接近：`KILL_BOUNDARY_ENERGY_STORY`。
