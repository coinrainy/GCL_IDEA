Idea Discovery Report: False-Negative GCL Toward BOND-GCL

方向：图对比学习假负样本方向，面向普通节点分类，优先考虑 2026 年论文潜力和实验提升可能性 日期：2026-06-26 Pipeline：research-lit → idea-creator → novelty-check → research-review → research-refine-pipeline 阶段结论：REVISE_TO_SMOKE_PLANNING

Executive Summary

本轮重新审视 false-negative GCL 方向后，不建议继续 DSR-GCL：已有 Cora seed=0 fix-audit-smoke 显示 DSR-full 明显低于 GRACE 与公平控制。新的推荐方向是 BOND-GCL / Basin-capped Objective for Negative Debiasing，它把问题从“每个 pair 是不是假负”改写为“同一语义盆地在 denominator 中被重复计数并产生过量排斥”。

fresh gcl_scientific_reviewer 对 BOND 给出 REVISE、novelty 6.0/10、confidence 0.74。结论是：BOND 有可救的新颖性空间，但不能直接实现 full pilot；必须先把方法压成单一机制 basin-level negative mass aggregation，并用反事实实验打掉 “只是 pair reweighting / E2Neg-style sampling” 的解释。

Literature Landscape

现有假负样本 GCL 文献已经覆盖了几条主路：

  • ProGCL / AUGCL：从 hard-negative mining 失败出发，估计 true-negative probability 或 affinity uncertainty。
  • KDD 2024 Node Similarity：把相似节点纳入更接近 ideal objective 的 soft target。
  • E2Neg：反对大规模 negatives，强调少量有效 negatives / cluster-center 风格采样。
  • SPGCL 2026：指出 message passing 会导致 positive pre-alignment，使正样本学习信号变弱。
  • Khan-GCL：生成 semantic hard negatives，但换 KAN encoder 且复杂度较重。

这些 prior 意味着：普通 pair reweighting、双阈值/interval、small-negative sampling、或生成式 hard negatives 都不是当前项目最优首选。更合理的切口是保留 GRACE/GCA 的轻量性，在 loss 聚合层做可证伪改动。

Ranked Ideas

Idea 1: BOND-GCL / Basin-capped Objective for Negative Debiasing

  • Method (what we actually do)：先为每个 anchor 构造一个低成本 anchor-local basin 划分；再把 InfoNCE denominator 从 pair-wise sum 改成 basin-wise negative mass aggregation；每个 basin 的 repulsive mass 有上限；overflow / boundary replacement 作为可关闭模块单独消融。
  • Hypothesis：节点级 GCL 中 false negatives 的核心伤害之一是同一语义盆地被重复计入 denominator，导致类内或近类区域被过度推远。
  • Minimum experiment：Cora seed=0 smoke，GRACE baseline + pair reweight proxy + E2Neg-style center + random basin + BOND-cap-only + overflow-only。
  • Expected outcome：BOND-cap-only 至少不低于 GRACE，并在 offline false-negative mass exposure、denominator effective number、uniformity 上优于 pair/random controls。
  • Novelty：6.0/10；closest prior 是 ProGCL、AUGCL、NodeSim、E2Neg、SPGCL。
  • Feasibility：中等；只改 loss / sampler，不换 encoder。
  • Risk：MEDIUM；最主要风险是数学上退化为 basin-level pair reweighting。
  • Contribution type：method + diagnostic。
  • Pilot result：SKIPPED；本轮未跑新实验。
  • Reviewer likely objection：Basin cap 可能只是 shared pair weight，overflow 可能才是真正提升来源。
  • Why we should do this：它比 DSR 更贴近 GRACE/GCA 插件，工程上更可能快速看到信号；同时比单纯 pair posterior 多一个可审查的 distribution-level failure mode。

Idea 2: PEARL-GCL

  • Method：用 Dirichlet-energy / positive informativeness 构建 positive signal，再用 low false-negative-risk local negatives 做对比。
  • Novelty：5.7/10。
  • Status：backup。受 SPGCL + ProGCL 夹击严重，除非 BOND smoke 失败且 SPGCL 差异化查新通过，否则不主推。

Idea 3: CNG-GCL v2

  • Method：使用 anchor-local conflict graph 选择低冲突、多 basin 覆盖的小 denominator。
  • Novelty：5.2/10。
  • Status：backup / ablation。太接近 E2Neg 或 diversity small-negative set，不建议作为主论文贡献。
