Executive Summary

CAST、BEACON、SPECTRA、PACER 的共同失败说明：把问题继续定义为“识别或修正假负样本”已经不够。低 overlap pair、高 label agreement pair、更好 rank/boundary diagnostics、甚至 train-label probe margin 改善，都没有稳定转化为 frozen Logistic Regression 的节点分类 accuracy。

新的问题定义应改成：

Node-node contrastive denominator 本身在少标签节点分类协议下可能是错误训练任务；与其修补 false negatives，不如把 SSL 目标改成不依赖其他真实节点作为负样本、但仍保留类别可读出的 masked discriminative evidence。

Top1 选择：MEGA-GCL / Masked Evidence Group Assignment for Graph Contrastive Learning。Fresh reviewer verdict 是 REVISE，不是直接 GO：MEGA 是当前列表里最像机制级 pivot 的方向，但必须先进入 Cora seed0 kill-smoke，证明它不是 GraphMAE / GraphMAE2 / GCMAE / VQ-codebook / SwAV-prototype / token corruption 的旧路线。

机制解释：这些方法都在优化“代理局部性质”：pair 选择、pair gate、embedding spectrum、probe margin。节点分类 accuracy 则由少量 train labels 能否稳定读出 class-granularity evidence 决定。若 SSL 目标改变的是全局几何或局部 margin，但没有让表示保留“哪些 ego evidence 对类别可判别”，就可能提升诊断而不提升 accuracy。

Ranked Ideas

1. MEGA-GCL / Masked Evidence Group Assignment — Top1

Core mechanism：构建 label-free evidence tokenizer，把每个节点 ego graph 的证据分为 feature groups、structural role groups、local residual groups。训练时 mask 掉一部分 ego evidence，encoder 需要从剩余上下文预测 masked evidence 所属 group。InfoNCE 的 negatives 是 evidence groups/token corruptions，不是真实节点，因此绕开 node-node false negatives。

Why it addresses no-go evidence：MEGA 不再问“哪个节点是假负样本”，也不再优化 rank/margin proxy；它直接训练表示保留可被下游线性读出的 ego evidence。若 false-negative 修正失败是因为 node-node denominator 与少标签分类错配，MEGA 是 objective-level 替换。

Novelty risk：高。Reviewer 明确指出最近邻包括 GraphMAE、GraphMAE2、GCMAE、VQ/codebook autoencoder、SwAV/PCL/PGCL/LR-GCL。必须强调不是 reconstruction，不是 latent remask prediction，不是 MAE+CL stacking，不是 prototype assignment；MEGA 的 claim 只能是“masked ego evidence group discrimination 是否比这些强 control 更适合 label-scarce node classification”。

Feasibility：高。Cora seed0 smoke 可复用 GRACE encoder、split、LogReg evaluator。Evidence groups 可由无标签 feature quantization + degree/ego residual buckets 生成。

Minimum kill-smoke：

  • M0 Cora seed0 only。
  • Compare GRACE、GraphMAE-style reconstruction、GraphMAE2-lite latent/remask prediction、GCMAE-style MAE+CL hybrid、VQ/codebook autoencoder、SwAV/prototype assignment、MINT token-corruption contrast、MEGA feature-only、structure-only、degree-only、full、random/permuted groups、matched-parameter GRACE。
  • Output accuracy、evidence prediction accuracy train/val/test-free、effective rank、alignment/uniformity、class evidence separation diagnostic after training。
  • No test labels for threshold/pretext/evidence groups。

2. GRANITE-GCL / Granularity-Preserving Prototype Bottleneck

Core mechanism：学习 balanced prototypes 作为 representation compression bottleneck，但不把同 prototype 节点当 positive。训练目标是 same-node view consistency + prototype usage balance + local evidence preservation，避免大簇吞掉少数类。

Novelty risk：中高。容易撞 PCL/cluster contrast/prototype SSL。差异点必须是 prototype 只作 bottleneck/augmentation constraint，不作 pseudo-positive mining。

Feasibility：中高。需要实现 prototype assignment、entropy balance、collapse guard。

Smoke：Cora seed0，GRACE / negative-free RR / prototype-as-positive control / GRANITE full / random prototypes / no-balance。

3. MINT-GCL / Masked Neighborhood Token Contrast

Core mechanism：把 ego neighborhood 分成匿名 token 序列，如 degree bucket、feature hash、edge-drop residual、hop role；mask 后预测 token type/order。负样本是 token corruption，不是节点。

Novelty risk：中。与 masked token prediction、Graph-BERT/GraphMAE 类似。

Feasibility：高。比 MEGA 更简单，但可能创新性略弱。

Smoke：Cora seed0，feature-token only / structure-token only / combined / shuffled tokens。

4. FOCUS-GCL / Fold-Consistent Objective Selection

Core mechanism：预注册多个 SSL objectives，用 train-only internal CV 或 validation 选择 objective / early stopping，不做 per-anchor routing，不把选择结果写回节点训练目标。

Novelty risk：高。可能只是 model selection / AutoML protocol。

Feasibility：高。工程上直接复用现有 runners。

Smoke：Cora seed0，比较 GRACE、GraphMAE-like、negative-free RR、MEGA；只允许 train/val 选 objective，然后报告 test。

5. EDGECASE-GCL / Evidence-Preserving Boundary Case Augmentation

Core mechanism：不使用标签 probe，而用无标签 evidence dropout sensitivity 找到低度、属性稀疏、结构歧义节点；对这些节点构造 evidence-preserving views，让表示保持局部可判别证据。

Novelty risk：中高。可能被说成 augmentation policy tuning。

Feasibility：中。需要预注册 evidence sensitivity，避免变成 PACER 的无标签版本。

Smoke：Cora seed0，random augmentation / sensitivity augmentation / no-edge sensitivity / no-feature sensitivity。

6. CLASSIC-GCL / Class-Granularity Invariance by Compression

Core mechanism：通过 multi-resolution compression 让 representation 不被单一大簇或过度 uniformity 主导；不使用 prototypes 作为 positives，只约束不同 resolution 下的信息保留。

Novelty risk：中。与 redundancy reduction / clustering / information bottleneck 接近。

Feasibility：中。需要 careful collapse diagnostics。

Smoke：Cora seed0，negative-free baseline / compression-only / multi-resolution full / random resolution。

7. SPLIT-MAE-CL / Split-Stable Masked Contrastive Hybrid

Core mechanism：训练 masked objective 与 contrastive consistency 的 hybrid，但只保留在 train/val split-stability 上一致的 pretext。不是 per-anchor routing，而是 dataset-level objective selection。

Novelty risk：高。可能像 protocol engineering。

Feasibility：高。适合做 backup 或 meta-baseline，不建议 Top1。

Smoke：Cora seed0，fixed hybrid / selected hybrid / random selected hybrid / GraphMAE-like。

Top1 Rationale

选择 MEGA-GCL 的原因：

  1. 它最大幅度避开四个 no-go 的共同失败结构：不再把真实节点作为 denominator negatives。
  2. 它仍然保持 GCL 风格：有 masked views、有 discriminative contrastive prediction、有 frozen encoder + LogReg 下游。
  3. 它比普通 GraphMAE 更贴近节点分类：目标不是重建原始属性，而是预测可压缩、可判别的 evidence group。
  4. 最小 smoke 很清楚：如果 MEGA full 不超过 GraphMAE-like reconstruction、random evidence、shuffled evidence target 和 GRACE，就直接 kill。
