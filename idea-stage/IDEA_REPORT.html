Executive Summary

CAST Certificate 的关键信号是“C4 低 overlap 但不提升”：它确实不是 kNN/PPR/CAST 的同一套 pair mining，却仍然没有带来稳定节点分类收益。因此下一条主线不应继续找更多跨节点 positives，而应研究 false-negative edit 的 conditional eligibility：一个疑似假负 pair 只有在不破坏线性分类边界、representation rank 和局部 uniformity 时才允许改变训练几何。

推荐 Top1 是 BEACON-GCL / Boundary-Eligibility Aware Contrastive Neutralization。它把 false-negative 修正从“相似 pair 该不该拉近”改成“这个 pair edit 对下游边界是否无害且有用”，用 counterfactual utility gate 决定 neutralize / keep-negative / defer。最小 smoke 不直接重训大模型，而是在 frozen GRACE embedding 上做 post-hoc relation edit sanity，先验证 eligibility gate 是否能筛掉 CAST 失败中的有害低-overlap pairs。

CAST Failure Brief

What Failed

  • Cora 3 seeds：C5 82.73±2.94，低于 kNN 84.23±1.16 与 CAST proxy 83.79±2.14。
  • CiteSeer 3 seeds：C5 71.77±0.09，低于 CAST proxy 71.99±0.36。
  • PubMed seed0：C5 85.47，低于 kNN 85.67 与 CAST proxy 85.62。
  • C4 的 overlap 诊断是有价值的：例如 PubMed seed0 C4 vs kNN/PPR/CAST overlap 为 0.0865 / 0.0422 / 0.0885。

Mechanistic Reading

CAST 证明了“找到不同的 pair”不等于“找到有任务价值的 pair”。低-overlap certificate positives 可能满足局部语义或 ego-prediction consistency，但它们仍可能跨越分类边界、压缩类间 margin，或削弱 InfoNCE 的 uniformity。换句话说，false-negative correction 的主要失败模式不是 novelty 不够，而是 edit utility 未被认证。

New Hard Constraints

  1. 不继续 CAST certificate / C4 / C5 的小修小补。
  2. 不把相似度、PPR、CAST、kNN 或 score fusion 当主贡献。
  3. 不把所有疑似假负直接转正。
  4. 每个候选必须解释“为什么 false-negative 修正会伤害/不提升节点分类”。
  5. 首轮只允许 smoke/pilot；默认 Cora/CiteSeer/PubMed，seeds 0-2。

Literature Landscape

已有图 false-negative 文献主要沿五条线展开：过滤、降权、转正、生成 hard negatives、概率建模。ProGCL 类方法强调图上 high-similarity negatives 常是假负；Node Similarity / positive mining 类方法强调扩展 positives；AUGCL / uncertainty HNM 强调 uncertainty；PU / debiased CL 强调 negative contamination。CAST 的负结果提示：这些线如果只回答“哪个 pair 像假负”，仍然不够；还必须回答“这个 pair edit 是否保留 downstream boundary”。

本轮把 gap 从 false-negative detection 收缩为：

False-negative edit eligibility：在节点分类 GCL 中，一个疑似假负 pair 只有当其编辑不会破坏分类边界、不会过度损伤 uniformity、且能减少有害 repulsion 时，才应被 neutralized 或 soft-positive。

Ranked Ideas

1. BEACON-GCL: Boundary-Eligibility Aware Contrastive Neutralization — Recommended

Core question：为什么低-overlap false-negative positives 仍然伤害？因为它们没有通过 boundary-utility gate。

Mechanism：对候选 pair (i, j) 不直接转正，而是做三路 eligibility：

  • benefit(i,j)：当前 negative repulsion 是否与多视图稳定性冲突；
  • boundary_risk(i,j)：虚拟 pair edit 是否降低 train-only / val-calibrated probe margin，或降低无标签 local boundary proxy；
  • geometry_risk(i,j)：虚拟 edit 是否损伤 representation rank、local uniformity 或 degree/class-balance proxy。

只有 benefit - lambda * boundary_risk - mu * geometry_risk > tau 的 pair 被 neutralized；未通过者继续作为普通 negatives；高风险相似 pair 被标记为 boundary-bearing negatives。

Why it is not CAST/kNN/PPR mining：候选池可以包含 kNN/PPR/CAST/C4，但贡献是 counterfactual acceptance test；同一候选池下必须比较 all candidates, random gate, shuffled utility, no-boundary gate。

Novelty risk：中等。它会被审稿人拿来和 ProGCL、AUGCL、Node Similarity、positive mining、validation-tuned mining 比较。差异点是 pair edit 的 downstream boundary utility，而不是 pair similarity/reliability 本身。

Feasibility：高。可以先复用现有 CAST runner 的 frozen embeddings、candidate relations、LogReg evaluator，并新增 post-hoc virtual edit gate。

Minimal smoke：Cora seed0，GRACE frozen embedding；候选池为 kNN ∪ PPR ∪ CAST proxy ∪ C4；比较 GRACE、kNN、CAST、C4、C5、all-candidate neutralization、BEACON gate、shuffled gate、no-boundary gate；输出 accuracy、label agreement、accepted-pair overlap、rank/uniformity delta、probe-margin delta。

2. BRI-GCL: Boundary-Residual InfoNCE

Core question：false-negative 修正伤害，是因为把 boundary-sensitive 维度也一起拉平。

Mechanism：把 embedding 分成 boundary-sensitive subspace 与 nuisance subspace。疑似假负只在 nuisance subspace 中 neutralize repulsion；boundary subspace 保留 contrastive separation。无标签版用 local covariance / spectral directions 近似 boundary；半监督版用 train-only probe 的 weight directions。

Novelty risk：中等偏高。与 disentanglement、subspace contrastive、supervised probe regularization 有接近点。

Feasibility：中等。实现比 BEACON 多一个 subspace decomposition，但可以先 post-hoc 测试。

Minimal smoke：Cora seed0，固定 embedding；对 C4/C5 pairs 做 full-space smoothing vs nuisance-only smoothing；若 nuisance-only 明显优于 full-space，说明 CAST 失败来自 boundary-dimension damage。

3. TRIAGE-GCL: Three-way False-negative Action Routing

Core question：false-negative correction 失败，是因为所有疑似假负被迫进入同一种动作。

Mechanism：每个 candidate pair 被路由到 positive, neutral, hard-negative, defer 四类。路由依据不是相似度，而是 edit utility、uncertainty、geometry damage。主贡献是 action routing，不是 pair discovery。

Novelty risk：中等。容易被认为是 CER/eligibility routing 的变体，必须强调 action utility 和 CAST 负结果驱动。

Feasibility：高。可以在现有 relation smoothing 框架中实现。

Minimal smoke：Cora/CiteSeer seed0；比较 positive-only、neutral-only、hard-negative-preserve、full routing。

4. RAMP-GCL: Repulsion Audit Memory for Persistent Harm

Core question：单次相似度或 certificate 容易误判；真正有害的假负 repulsion 应该在训练中持续出现。

Mechanism：维护 pair-level 或 anchor-level repulsion conflict memory。只有长期稳定地表现为“高相似 + 多视图一致 + repulsion force 大 + geometry risk 低”的 pair 才被 neutralized。

Novelty risk：中等。会与 curriculum false-negative filtering / IFND 比较。

Feasibility：中等。需要改训练 loop，存储 top-k memory。

Minimal smoke：Cora seed0，warmup 后每 20 epoch 更新 memory；比较 one-shot gate vs persistent gate。

5. QUOTA-GCL: Uniformity-budgeted False-negative Neutralization

Core question：false-negative 修正伤害，是因为局部正样本扩张吞掉了 uniformity budget。

Mechanism：每个 anchor、degree bucket、prototype bucket 都有 neutralization quota。候选 pair 的接纳不仅看 semantic signal，还看是否超出 uniformity / rank budget。

Novelty risk：中等偏低。可能被认为是 conservative positive mining + quota。

Feasibility：高。适合做 BEACON 的 ablation 或 safety component。

Minimal smoke：在 C4/C5 候选上加 quota，比较 no quota / degree quota / prototype quota。

6. INV-GCL: Invariance-before-Debias View Training

Core question：跨节点假负修正不稳定，可能说明节点自身增强视图还没学到足够稳定的 invariance。

Mechanism：先训练 anchor-local invariance model，只有当 anchor 自身视图稳定后才允许跨节点 false-negative edit。核心不是找 pair，而是先校准 anchor 是否 eligible。

Novelty risk：中等。接近 curriculum / view stability / GraphMAE-style pretext。

Feasibility：中等。需要训练阶段 schedule。

Minimal smoke：Cora seed0，比较 no warmup、fixed warmup、anchor-stability-gated warmup。

Eliminated or Deprioritized Directions

  • CAST certificate / C4 / C5 continuation：Pilot-A 已给出 no-go。
  • Pure kNN/PPR/CAST positive mining：无法解释低-overlap无收益。
  • Loss-only reweighting：创新性不足，且容易回到 ProGCL/DCL 线。
  • Direct all-positive conversion：CAST/IRIS/CPR 失败已经证明误转正风险高。
  • Heavy generators / KAN / LLM views：工程复杂且会混入架构贡献，不适合当前最小 smoke。
