## 0. Low-Novelty Counterexample: IGT-GCL

IGT-GCL / Interval-Guarded Pair Targeting 当前 novelty score 为 `5.5/10`，decision 为 `REVISE`。本轮把它作为低新颖性反例：

- 它仍在 point posterior / interval / abstention / sample reweighting 邻域。
- reviewer 认为其可保留新意只在 lower/upper-bound 三分区 target construction。
- 致命风险是 `[l,u]` 像两个阈值或两个 score，而不是可验证统计区间。
- 若 full interval 不能击败 midpoint posterior、NodeSim/IFL-style point target、only downweight、only expansion，则贡献不成立。

本轮候选必须避开：给 pair 估计单点概率、区间概率、ambiguous abstention、普通 negative downweight、普通 positive expansion、matched sparsity mask。

## 1. Updated Literature Boundary

近期检索增加了几条强约束：

- **SPGCL, 2026**：指出 GCL positive samples 可被 message passing pre-alignment trivialize；用 Dirichlet energy 分离传播并用低能量特征做 positive sampling。
- **ASPECT, 2026**：节点级自适应 spectral fusion，低/高频 view 不同节点有不同偏好。
- **FC-GSSL, WWW 2026**：高频信号有价值，但高频局部性和过度依赖会影响泛化；通过 frequency-corrupt 自监督融合高低频。
- **ReGCL, AAAI 2024**：从 InfoNCE 梯度角度分析 GNN message passing 与 contrastive distinction 的冲突，并做 gradient-guided structure learning / gradient-weighted InfoNCE。
- **GraphRank**：用 rank loss 避免 InfoNCE 把负样本相似度无限压低，从而缓解 intra-class negatives。
- **Zero-Mean Regularized Spectral CL, ICLR 2024**：在 spectral contrastive loss 中放松负对正交性，缓解 wrong connections。

因此，新的主 idea 不能只是“修正 pair 权重”，而应改造 **InfoNCE 作用的表示子空间 / 梯度通道 / 训练目标结构**。

## 3. Top 1: SGF-GCL

**Problem Anchor**  
节点级 GCL 的 false negative 问题不仅是“哪个 pair 被误当负样本”，更是 **InfoNCE 的 repulsive gradient 被允许作用在整条表示向量上**。当同类或同语义节点被当作负样本时，denominator 会在 class-smooth / low-frequency 子空间中撕裂本该服务于节点分类的流形。

**Method Thesis**  
把表示拆为两个互补通道：

- `z_s`: class-smooth channel，低频/低 Dirichlet energy，承载节点分类中常见的平滑类别流形；
- `z_r`: boundary-residual channel，高频/高 Dirichlet energy，承载局部边界、异配边和 instance-level 区分信号。

SGF-GCL 只让 InfoNCE 的负样本 repulsive gradient 作用于 `z_r`，而 `z_s` 使用 negative-free consistency + variance/covariance regularization 学习稳定流形。最终 frozen embedding 使用 `[z_s || z_r]` 或 validation-selected fixed concat，但禁止 test 选择。

**Core Delta**  
不是判断某个 negative 是真负还是假负，而是改变 repulsion 可以伤害的表征方向：

```text
false negatives may still exist,
but they cannot tear apart the class-smooth subspace.
```

## 4. Top 2 Backup: CMO-GCL

若 reviewer 认为 SGF-GCL 与 SPGCL/ASPECT 太近，可退到更几何化的 CMO-GCL：

- 用 self-supervised low-rank manifold loss 学 `U`；
- 将 contrastive residual `z_r = (I - UU^T)z` 限制在 manifold orthogonal complement；
- class manifold 用 Barlow/VICReg-style negative-free 维护非塌缩；
- 直接诊断 InfoNCE repulsive gradient 与 `U` 的投影大小。

该方向更清楚地强调 **gradient orthogonalization**，但实现比 SGF 稍复杂。

## 5. Must-Avoid Variants

以下不是新 idea，只能做 baseline / ablation：

- point posterior GCL；
- interval / ambiguity-set GCL；
- false-negative filtering；
- negative downweight；
- positive expansion；
- no-abstention / abstention variants；
- rank loss alone；
- spectral fusion alone；
- low-rank regularization alone；
- GraphMAE + GRACE 简单加权 hybrid。

## 6. Required Novelty Check for SGF-GCL

SGF-GCL 必须与以下 prior 明确区分：

| Prior | Overlap | Required delta |
|---|---|---|
| SPGCL 2026 | Dirichlet energy, positive learning trivialization, energy-aware propagation | SGF 的主目标是 false-negative damage containment；负样本 repulsion 被限制在 residual channel，而非重新构造 positive sampling。 |
| ASPECT 2026 | low/high frequency views, node-wise spectral fusion | SGF 不是为每个节点融合频率 view，而是为不同 loss/gradient 分配可作用的频谱子空间。 |
| ReGCL 2024 | message passing and InfoNCE gradient conflict | SGF 不做 gradient-weighted InfoNCE 或 structure learning，而做 subspace-level gradient firewall。 |
| GraphRank | 负样本不再无限推远 | SGF 仍可保留 InfoNCE/rank 对 residual 的边界压力，但保护 class-smooth subspace。 |
| Zero-Mean Spectral CL | 放松 negative pair orthogonality | SGF 是图频谱和节点分类导向的 channel routing，不是全局 zero-mean regularizer。 |
| LR-GCL / GCL-LRR | low-rank graph contrastive learning | SGF 的低频通道不是简单低秩正则，核心是 repulsive-gradient containment。 |
