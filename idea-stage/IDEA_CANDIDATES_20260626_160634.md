## Shared Failure Explanation

三条 no-go 指向同一个问题：我们一直在优化 downstream evaluator 的代理变量。

- CAST 优化的是 pair novelty / low overlap。
- BEACON 优化的是 pair purity 与局部几何安全。
- SPECTRA 优化的是 representation diagnostics。

这些都不是最终 frozen Logistic Regression 在少标签 `1:1:8` 下真正优化的目标。节点分类 accuracy 取决于少量 train labels 能否在 frozen representation 上学出稳定、可泛化的线性类边界。下一轮 idea 必须从“pair 是否正确”或“diagnostic 是否好看”转向“pretraining objective 是否让少标签 linear readout 更容易”。

## Top1

选择 **PACER-GCL**。

它最直接吸收三条失败证据：不再把 pair purity、overlap、rank/boundary retention 当作目标，而是显式检查 objective 是否帮助少标签 linear probe 的可读出性。为避免违反约束，PACER 不做 cross-node pair editing；它是 **anchor-level objective routing + masked logit-preserving consistency**，并且必须诚实标注为 semi-supervised / train-label-calibrated GCL，而不是纯 unsupervised GCL。

## Top1 Kill Rules

- PACER full 不超过 GRACE，或不超过 probe-only/mask-only controls：`KILL_PACER`。
- shuffled-label probe 或 random-probe route 接近 PACER：`KILL_PROBE_ALIGNMENT_STORY`。
- scalar loss-weight control 接近 PACER：说明只是 reweighting，`KILL_OBJECTIVE_ROUTING_STORY`。
- train accuracy/margin 提升但 validation/test 下降：`KILL_OVERFIT_TO_TRAIN_PROBE`。
- 只有使用 validation labels 训练才有效：`INVALID_PROTOCOL`。
