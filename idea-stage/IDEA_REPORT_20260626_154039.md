## Failure Evidence

### CAST Pilot-A

CAST 的 C4 latent certificate 在 Cora/CiteSeer/PubMed seed0 上与 kNN/PPR/CAST overlap 都较低，尤其 PubMed seed0 与 kNN/PPR/CAST 的 overlap 分别约为 `0.0865/0.0422/0.0885`。这说明 C4 不是简单复制 kNN/PPR/CAST mining。

但 C4/C5 没有稳定超过最强 mining controls。Cora 与 CiteSeer 上 kNN/CAST proxy 仍更强；PubMed seed0 上 C4/C5 也没有形成清晰优势。因此“pair 选择足够不同”不是充分条件。

### BEACON BE-M0

BEACON B5 full gate 在 Cora seed0 上有最高 label agreement `0.8611`，probe-margin delta `0.3063`，rank delta 也比若干候选更温和。它确实选出了 offline 看起来更安全的 pair。

但 B5 accuracy `84.87` 低于 kNN `85.29`、CAST proxy `85.70`、all-candidate neutralization `85.42`，也低于 shuffled/no-boundary/no-geometry variants。这说明“更高 label agreement + 更好 margin/rank diagnostics”仍不是充分条件。

## Mechanism Explanation

pair-level false-negative correction 失败的关键，不是当前证书不够复杂，而是它优化了错误层级。

### 1. Pair correctness cannot guarantee boundary usefulness

节点分类的线性 evaluator 需要的是一组全局可分方向。把某些同类或近似同类 pair 拉近，可能提升局部 agreement，却同时压缩类内多簇、少数类边界和高频残差信号。对 Cora/CiteSeer 这类文本属性图，类别内部可能本来就有多个 topic submodes；过度 smoothing 会让 Logistic Regression 更难用少量 train labels 找到清晰边界。

### 2. False-negative repair may trade away effective rank

relation smoothing / neutralization 往往会减少 repulsion，但 repulsion 也承担 uniformity 和 rank 维持。若修正 false negatives 的同时降低有效秩，模型会得到更“干净”的局部 pair 诊断，却丢掉可线性分割的表示维度。

### 3. Offline pair metrics are not downstream objectives

label agreement 是 oracle diagnostic；margin-rank 是局部 geometry diagnostic；overlap 是 novelty diagnostic。它们都不是 downstream train/val LogReg 的目标。BEACON 的 no-go 说明：即使这些诊断变好，也可能只是让已容易分类的区域更一致，而没有帮助真正决定 accuracy 的边界节点。

### 4. Pair edits happen too late

如果 GNN message passing、augmentation 或 InfoNCE 训练已经把边界频带抹平，后续再用 certificate pair 去修 denominator/numerator，只是在一个已损伤的 representation 上做关系补丁。更合理的方向是从一开始约束 representation 的边界能量、冗余和有效秩。

## New Problem Statement

不要继续问：

> Which cross-node pairs should be converted from negatives to positives/neutral pairs?

改问：

> Can a graph SSL objective avoid false-negative damage by preserving boundary-sensitive representation geometry without cross-node pair mining?

## Selected Top1: SPECTRA-GCL

**One-sentence thesis**：SPECTRA-GCL replaces pair-level false-negative correction with a negative-free redundancy-reduction objective that explicitly preserves label-free spectral boundary energy and effective representation rank, so false negatives are avoided by objective design rather than mined away pair by pair.

### Method Sketch

Given two graph augmentations, an encoder produces node representations `h1, h2` and projected representations `z1, z2`.

SPECTRA uses three terms:

1. `L_rr`：Barlow/VICReg/CCA-style negative-free redundancy-reduction loss on same-node two-view representations。
2. `L_boundary`：label-free boundary energy preservation。用 graph Laplacian residual / high-pass edge residual 近似边界频带，约束增强前后节点表示在中高频 band 的能量分布不被过度压平。
3. `L_rank`：effective-rank / variance guard，防止 negative-free objective collapse 或过度低秩。

总目标：

```text
L = L_rr + lambda_b * L_boundary + lambda_r * L_rank
```

其中 `lambda_b/lambda_r` 在 smoke 中固定为小集合，不使用 test set 选择。正式前只能通过 train/val 或预注册 protocol 决定。

### What It Is Not

- 不是 kNN/PPR/CAST mining。
- 不新增跨节点 positives。
- 不做 candidate relation smoothing。
- 不做 score fusion。
- 不把 label agreement 当训练目标。
- 不继续 CAST C4/C5 或 BEACON gate。

### Novelty Boundary

SPECTRA 必须诚实承认与以下方向接近：

- Graph Barlow Twins / CCA-SSG / BGRL：negative-free GCL。
- VICReg / Barlow Twins：redundancy reduction 与 collapse guard。
- GraphMAE / masked graph modeling：非对比式图 SSL。
- spectral GCL / DSR-style spectral branch：谱信息与边界残差。

可争取的新意是：**从 false-negative pair-correction 失败出发，把问题重定义为 boundary-energy-preserving negative-free GCL，并用 CAST/BEACON no-go 的诊断证明 pair correctness 不是主瓶颈。**

- effective rank / participation ratio。
- covariance off-diagonal redundancy。
- alignment/uniformity for comparability, but not used as success alone。
- boundary energy retention：pre/post augmentation and pre/post training。
- high-pass residual variance by degree bucket。
- label-only offline boundary bucket accuracy, reported only as diagnostic and never used for training/threshold selection。
