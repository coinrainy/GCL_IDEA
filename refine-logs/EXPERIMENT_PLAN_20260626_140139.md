# Experiment Plan: CAST Certificate Pilot-A

**Problem**: 图对比学习节点分类中，false negatives 往往被普通 InfoNCE 当作 negatives；但简单 kNN/PPR/embedding positive mining 容易缺少语义判据。  
**Method Thesis**: CAST latent target-prediction certificate 用 label-free ego target prediction energy 认证跨节点候选关系，从而比普通 CAST proxy / kNN / PPR 更可靠地发现 false-negative positives。  
**Date**: 2026-06-26

## Claim Map

| Claim | Why It Matters | Minimum Convincing Evidence | Linked Blocks |
|-------|----------------|-----------------------------|---------------|
| C1: certificate improves relation quality and downstream accuracy | 主贡献必须不是普通 proximity mining；Cora smoke 中 C4/C5 已出现正信号 | 在 Cora/CiteSeer/PubMed seeds 0-2 上，C4/C5 平均 accuracy 不低于 CAST proxy，且 label agreement 高于 CAST proxy | B1, B2 |
| C2: certificate is not just kNN/PPR/CAST re-ranking | 审稿人最可能质疑它只是高级 positive mining wrapper | selected-pair overlap with kNN/PPR/CAST 不能过高；certificate score 对 label agreement 的 partial correlation 在控制 feature/embedding/PPR/degree 后仍为正 | B2, B3 |

## Paper Storyline

- Main paper must prove: CAST certificate 在小规模 Pilot-A 上有一致正信号，并且不是纯 kNN/PPR/CAST proxy。
- Appendix can support: certificate training loss、candidate energy distribution、失败样例。
- Experiments intentionally cut: Amazon/DBLP/Wiki-CS、多 seed formal、复杂超参 sweep、更多 IRIS/CPR 变体。

## Experiment Blocks

### Block 1: Pilot-A Main Result

- Claim tested: C1
- Why this block exists: 验证 clean Cora smoke 是否能在 Planetoid 三数据集和 3 seeds 下保持方向一致。
- Dataset / split / task: Cora, CiteSeer, PubMed；project stratified random `1:1:8`；节点分类；seeds `0,1,2`。
- Compared systems:
  - GRACE frozen embedding baseline
  - kNN positives
  - PPR/diffusion positives
  - CAST proxy
  - C4 latent target certificate
  - C5 certificate + CAST score
- Metrics:
  - primary: LogReg test@best-val accuracy
  - secondary: offline label agreement, false-negative mass after closure, runtime
- Setup details:
  - frozen GRACE warmup encoder per dataset/seed
  - certificate predictor trained label-free on ego summaries to predict frozen target embeddings of graph-neighbor targets
  - same positive budget for all relation methods
  - evaluator uses `logreg_val_eval`; `C` selected only by train/val
- Success criterion:
  - C5 mean accuracy >= CAST proxy mean on at least 2/3 datasets
  - C5 label agreement > CAST proxy on at least 2/3 datasets
  - no dataset shows catastrophic drop > 0.5 accuracy point versus CAST proxy
- Failure interpretation:
  - If only Cora improves, the smoke was likely dataset-specific.
  - If accuracy improves but label agreement does not, relation quality claim is weak.
- Table / figure target: Main paper Table 1 candidate, but only after formal protocol later.
- Priority: MUST-RUN

### Block 2: Novelty Isolation Diagnostics

- Claim tested: C2
- Why this block exists: 排除“只是 kNN/PPR/CAST proxy”的审稿风险。
- Dataset / split / task: same Pilot-A runs from Block 1.
- Compared systems: kNN, PPR/diffusion, CAST proxy, C4, C5.
- Metrics:
  - selected-pair overlap with kNN/PPR/CAST
  - partial correlation between certificate score and offline label agreement after controlling feature similarity, embedding similarity, PPR/diffusion proximity, and degree gap
  - certificate energy distribution for same-label vs different-label pairs, reported as diagnostic only
- Setup details:
  - labels must not enter relation selection
  - partial correlation computed offline after the run
  - overlap computed over selected directed pairs under equal positive budget
- Success criterion:
  - C4/C5 overlap with kNN not excessive; target `<0.75`
  - C4/C5 overlap with CAST proxy not trivial; for C4 target `<0.5`, for C5 allowed higher because it intentionally mixes CAST
  - certificate partial correlation positive on at least 2/3 datasets
- Failure interpretation:
  - High kNN overlap plus no partial correlation means the certificate is decorative.
- Table / figure target: Figure 2 diagnostic panel or appendix table.
- Priority: MUST-RUN

### Block 3: Simplicity Check

- Claim tested: C1 and C2
- Why this block exists: 确认证书 + CAST 分数已经足够，不需要回到 IRIS/CPR 大量机制。
- Dataset / split / task: only Cora seed 0-2 unless Block 1 is strongly positive.
- Compared systems:
  - C4 latent target certificate
  - C5 certificate + CAST score
  - CPR I17 historical smoke result, not rerun unless needed
- Metrics: accuracy, label agreement, overlap.
- Setup details: no new model component; use existing runner outputs where possible.
- Success criterion: C5 dominates C4 on accuracy without much label-agreement loss; otherwise keep C4 as simpler method.
- Failure interpretation: if C5 only wins by proximity mixing, paper should focus on C4.
- Table / figure target: Appendix or method selection paragraph.
- Priority: NICE-TO-HAVE

## Compute and Data Budget

- Total estimated GPU-hours: `2-4` for Pilot-A, depending on PubMed certificate scoring.
- Data preparation needs: reuse PyG Planetoid datasets and saved project split JSON.
- Human evaluation needs: none.
- Biggest bottleneck: saving/reusing certificate score matrices for partial-correlation diagnostics without excessive memory.

## Risks and Mitigations

- Risk: C5 improvement is only Cora-specific.
- Mitigation: require 2/3 Planetoid datasets to pass before any broader pilot.

- Risk: certificate reduces to kNN/PPR mining.
- Mitigation: require overlap and partial-correlation diagnostics; keep kNN/PPR controls in every run.

- Risk: result JSON says `project_dirty=true` because logs are created during runs.
- Mitigation: commit runner/config before Pilot-A; record commit hash and explain log-induced dirty flag if it persists.

- Risk: PubMed dense pairwise matrices exceed memory.
- Mitigation: use candidate-pool scoring and sampled pair diagnostics for PubMed if needed.
