# EXPERIMENT_PLAN: IGT-GCL

方法：IGT-GCL / Interval-Guarded Pair Targeting for Graph Contrastive Learning  
阶段：pilot planning only  
正式决策：**REVISE**  
当前禁止：不跑 full formal，不做性能 claim，不混 split。

## 1. Experiment Goal

本计划的目标不是证明 SOTA，而是回答一个更小、更硬的问题：

> 在节点级 GCL 中，lower/upper-bound interval target 是否真的比点式 posterior、普通扩正、普通降权和随机稀疏化更能处理 false-negative / hard-negative ambiguity？

若答案是否定的，IGT-GCL 应被 pivot 或 kill。

## 2. Protocol

必须遵守项目协议：

- 同配图：stratified random `1:1:8` split。
- 异配图：官方 fixed split。
- 所有方法使用相同 split 文件、相同 seed 列表。
- GCL 表示学习后统一 frozen encoder + Logistic Regression evaluator。
- Logistic Regression 参数只能由 train/val 或 train-only CV 选择。
- pilot 建议 seeds `0,1,2`；formal 若未来批准，默认 seeds `0-9`。
- pilot/development 结果只能用于 go/revise/pivot/kill，不支持论文 claim。

## 3. Claim-to-Evidence Map

| Claim ID | Claim | Needed evidence | Decision rule |
|---|---|---|---|
| C1 | interval pair target 有诊断意义 | high-`l` same-label precision、low-`u` true-negative precision、ambiguous false-negative enrichment | 若不优于 point posterior top-k / random matched mask，则 REVISE/KILL |
| C2 | interval 比点式 posterior 更稳 | full IGT-GCL > midpoint posterior and NodeSim/IFL-style point posterior on pilot mean | 若差距小或反向，则不能声称 interval 是核心 |
| C3 | three-way action 必要 | full > only positive expansion, only negative downweight, no abstention | 若任一简单变体匹配 full，则降级贡献 |
| C4 | 不只是稀疏化或预算效应 | full > random interval with matched sparsity and same pair count | 若 matched random 接近，则 REVISE/KILL |
| C5 | 无明显 collapse | alignment/uniformity、embedding rank、class-wise/degree-wise diagnostic 正常 | 若 rank collapse 或 minority degradation 明显，则 KILL or restrict |

## 5. Variants

所有 variant 共享 encoder family、augmentation、training budget、split、seed 和 evaluator。

| Variant | Description | Required |
|---|---|---|
| V0 vanilla GRACE | 当前最小 GCL baseline | yes |
| V1 vanilla GCA | 若本地 GCA 适配可用，则作为 augmentation-aware baseline | preferred |
| V2 point posterior | 用单点 `p_ij` 做 soft target，贴近 NodeSim/IFL-style | yes |
| V3 midpoint target | `p_ij = (l_ij + u_ij) / 2` | yes |
| V4 ProGCL-style true-negative weighting | 只用 true-negative probability 权 denominator | yes |
| V5 only positive expansion | 只把 high-`l` 放 numerator，不改 denominator | yes |
| V6 only negative downweight | 只低权或移除 high-`u` ambiguous negatives | yes |
| V7 no abstention | ambiguous pair 被强制归入点式 target | yes |
| V8 random interval matched sparsity | 与 full 相同 `|P_i|, |H_i|, |A_i|`，但随机选 pair | yes |
| V9 full IGT-GCL | lower/upper target + reliable hard negatives + abstention | yes |

## 6. Diagnostics

### Pair Quality

标签只用于离线诊断，不参与 pair construction：

- `precision_high_l`: `P(y_i = y_j | j in P_i)`。
- `precision_low_u_true_negative`: `P(y_i != y_j | j in H_i)`。
- `fn_enrichment_ambiguous`: ambiguous set 中 same-label pair 相对普通 negatives 的富集倍数。
- `pair_action_size`: per-anchor `|P_i|`, `|H_i|`, `|A_i|` mean/std。
- degree bucket / class bucket precision，避免只服务大类或高 degree 节点。

### Representation Health

- downstream Logistic Regression val/test accuracy；
- alignment / uniformity；
- embedding covariance rank or effective rank；
- per-class accuracy；
- degree-bucket accuracy；
- training time and memory。

### Gradient Diagnostics

- denominator gradient mass by pair set；
- hard-negative gradient mass before/after interval guard；
- ambiguous stop-gradient 是否真的不产生 repulsive gradient。

## 8. Hyperparameter Discipline

Pilot may explore small grids but must record all tried values. Suggested fixed defaults:

- candidate top-k: `64` or `128`；
- warmup epochs: dataset default GRACE/GCA budget fraction, e.g. `10%-20%`；
- `tau_l`: chosen by target per-anchor positive quota, not test performance；
- `tau_u`: chosen by target reliable hard-negative quota, not test performance；
- `alpha`: small soft-positive mass, e.g. `0.05-0.2`；
- `gamma`: default `0` for ambiguous stop-gradient removal；
- Logistic Regression `C`: train/val selected under project evaluator.

Search budget must be equalized across point posterior, only-downweight, only-expansion and full interval. If full IGT-GCL gets more tuning, this must be disclosed and cannot support formal claim.

## 9. Stop Rules

Kill or pivot if:

- full interval does not beat midpoint posterior on pilot diagnostics；
- random interval matched sparsity matches full interval；
- only downweight or only expansion matches full interval；
- ambiguous set is not enriched for false negatives；
- `P_i` high-l precision is low or strongly biased toward majority/high-degree classes；
- method causes rank collapse, severe uniformity loss, or consistent Amazon degradation；
- implementation requires a much larger model/search budget than baselines。
