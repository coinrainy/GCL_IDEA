# Research Proposal: CER-GCL — Contrastive Eligibility Routing for Graph SSL

## Problem Anchor

- Bottom-line problem: 在普通节点分类的图对比学习中，标准 GRACE/GCA 式预训练默认每个节点都适合强 positive alignment；我们需要验证并利用“哪些节点适合强 alignment，哪些节点不适合”这一节点级差异，而不能把它偷换成 boundary-aware GNN 或 false-negative mining。
- Must-solve bottleneck: 当前 idea 的最大瓶颈是 novelty collision。BCE-GCL 的 boundary 叙事已被 BES/UGCL/ASPECT/HAR 等近邻强烈包围；必须收缩为无监督 contrastive eligibility diagnostic + 最小 objective routing，并证明它不是 degree、uncertainty、augmentation strength 或普通 loss weighting 的代理。
- Non-goals: 不追求 SOTA claim；不做 LLM/KAN/大架构替换；不使用 test label 或 test set 调参；不声称 first boundary-aware GCL；不把 pilot/development 结果写成 formal claim。
- Constraints: 基于现有 GRACE/GCA 代码路线；节点分类主协议为 stratified random `1:1:8`；GCL evaluator 为 frozen encoder + Logistic Regression；首轮只允许 smoke/pilot/development 证据。
- Success condition: 纯无监督 eligibility signal 能预测节点对强 alignment 的受益/受损差异，并且 objective routing 在控制 degree/uncertainty/augmentation strength 与 scalar weighting 后仍有独立增益。

## Method Thesis

CER-GCL 将节点级 contrastive eligibility 定义为 anchor-level objective intervention：eligible anchors 使用完整 InfoNCE；ineligible anchors 不再优化 full denominator discrimination，而是使用 positive-only stop-gradient consistency。

## Core Objective

Let `z_i^a` and `z_i^b` be normalized embeddings for node `i` under two standard GRACE views, and `sg(.)` be stop-gradient.

Eligible anchor objective:

```text
L_i^NCE = -log exp(sim(z_i^a, z_i^b)/tau) /
          sum_j exp(sim(z_i^a, z_j^b)/tau)
```

Ineligible anchor objective:

```text
L_i^PO = 1 - sim(z_i^a, sg(z_i^b))
```

Mixed objective:

```text
L = mean_i [ m_i * L_i^NCE + (1 - m_i) * L_i^PO ]
```

This differs from scalar weighting, which keeps the same objective:

```text
L_weight = mean_i w_i * L_i^NCE
```

## Eligibility Gate

The gate is frozen after warmup and uses no labels:

```text
q_i = rank_norm(drift_i) + rank_norm(fs_residual_i)
m_i = 1[q_i <= quantile(q, rho)]
rho = 0.8
```

- `drift_i`: warmup-period multi-view positive drift, averaged over repeated standard augmentations.
- `fs_residual_i`: degree-residualized feature-structure disagreement.
- `s_loss` is not used in the gate; it is logged only as a diagnostic.

## Denominator Policy

Pilot default: ineligible nodes are removed as anchors from full InfoNCE but remain as stop-gradient denominator candidates for eligible anchors. A denominator-detach ablation must test whether negative-side gradients are still harming the intended group.

## Contribution Focus

- Dominant contribution: objective-type switching for node-level contrastive eligibility.
- Supporting contribution: falsification protocol separating routing from degree/uncertainty proxies and scalar weighting.
- Non-contributions: adaptive augmentation, spectral fusion, pair false-negative correction, supervised boundary shaping, new encoder.

## Required Stop Rules

- If same-score scalar weighting matches CER-GCL, kill routing claim.
- If degree/uncertainty controls explain the signal, kill eligibility claim.
- If positive-only branch collapses embedding rank/variance in smoke, stop before pilot accuracy interpretation.
- If only overall accuracy improves without bucket diagnostics, do not make mechanism claim.
