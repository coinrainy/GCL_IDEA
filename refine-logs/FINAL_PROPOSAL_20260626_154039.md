# Final Proposal: SPECTRA-GCL

- 时间：2026-06-26T15:40:39Z
- 阶段：post CAST/BEACON no-go idea proposal
- 状态：`GO_TO_SPECTRA_SMOKE_PLANNING`
- Claim status：proposal only；无 smoke/pilot/formal 结果；不支持性能 claim。

## Motivation

CAST 和 BEACON 的失败共同说明：pair-level false-negative correction 的 offline 指标可以变好，但节点分类 accuracy 不一定提升。

- CAST C4 低 overlap，说明不是普通 kNN/PPR/CAST mining；但低 overlap pair 没有稳定提升分类。
- BEACON B5 label agreement 与 margin-rank diagnostics 更好；但 accuracy 仍低于简单 controls。

这提示我们：当前瓶颈不是“还没找到更好的 pair”，而是 **pair edit 没有直接保护 downstream classifier 需要的表示结构**。节点分类需要可线性分割的边界、足够的有效秩、不过度平滑的中高频残差，以及合理的 alignment-uniformity 平衡。

## Core Idea

**SPECTRA-GCL / Spectral Boundary-Preserving Redundancy Reduction for Graph Contrastive Learning**

SPECTRA-GCL 放弃跨节点 positive/neutral mining，改用 negative-free same-node two-view SSL：

1. 用 redundancy-reduction objective 避免 InfoNCE denominator 中的 false-negative repulsion。
2. 用 label-free spectral boundary energy preservation 防止 negative-free alignment 把边界频带抹平。
3. 用 effective-rank / variance guard 防止 collapse 或过低秩。

## Objective

给定两个增强视图 `G1, G2`：

```text
h1 = encoder(G1)
h2 = encoder(G2)
z1 = projector(h1)
z2 = projector(h2)
```

### Redundancy Reduction

`L_rr` 使用 Barlow/VICReg/CCA-style objective：

- same-node two-view invariance；
- per-dimension variance lower bound；
- cross-dimension covariance decorrelation。

### Boundary Energy Preservation

用 graph Laplacian residual 近似边界频带：

```text
r(X) = X - Smooth_A(X)
```

其中 `Smooth_A` 可以是 row-normalized adjacency smoothing 或一阶/二阶 propagation。对表示 `h` 也计算同类 residual：

```text
r(h) = h - Smooth_A(h)
```

`L_boundary` 不用标签，不判断节点同类。它只要求训练后的表示不要把输入/早期表示中的中高频能量全部压平：

```text
E_boundary(h) = mean_i ||r(h)_i||_2
L_boundary = distance(normalized_bucket(E_boundary(h1)), normalized_bucket(E_boundary(h2)))
             + energy_floor_penalty(E_boundary(h1), E_boundary(h2))
```

首版 smoke 用 degree buckets 或 quantile buckets 统计，避免单节点能量噪声。

### Rank Guard

`L_rank` 约束表示 covariance 的 participation ratio / effective rank 不低于固定下界，或使用 VICReg variance term 的简化版本。

### Total

```text
L_SPECTRA = L_rr + lambda_b * L_boundary + lambda_r * L_rank
```

## Non-goals

- 不 claim new encoder。
- 不使用 test label 或 test set 调参。
- 不构造 cross-node positive/neutral pairs。
- 不做 kNN/PPR/CAST mining。
- 不做 CAST C4/C5 修补。
- 不做 BEACON gate threshold/weight search。
- 不把 smoke/pilot/development 结果称为 SOTA、robust 或 formal。

## Why This Could Work

如果 false-negative damage 的主要影响是过度 repulsion 或错误 attraction 导致 class boundary 被扭曲，那么移除 cross-node negatives 可以避免一部分错误 repulsion。但 negative-free alone 又可能 collapse 或过度 smooth。SPECTRA 的边界能量项就是为了解决这个二阶问题：不再问“哪个节点该当 positive”，而是让表示保留分类边界可能需要的局部残差和有效维度。

## Novelty Risk

| Prior family | Risk | Required control |
|---|---|---|
| BGRL / CCA-SSG / Graph Barlow Twins | negative-free 不是新贡献 | S1 negative-free baseline 必须强对照 |
| VICReg / Barlow Twins | variance/covariance guard 已有 | SPECTRA 的新增点必须来自 graph boundary energy |
| GraphMAE / MaskGAE | masked/reconstruction SSL 已覆盖非对比路线 | 本方法首版不做 decoder，避免变成 GraphMAE clone |
| spectral GCL / DSR | 谱域与边界残差已有近邻 | 不做 two-channel firewall；只做 boundary-energy preservation |
| BEACON/CAST | 可能被误读为换一个 diagnostic | 不使用 pair gate，不用 label agreement 做训练目标 |
