## Method Name

**GRAFT-GCL / Graph Prior-Task Fitted Contrastive Learning**

## Problem Anchor

本项目连续 no-go 表明：更好的 pair mining、更好的 representation diagnostics、更好的 train-label proxy、更好的 masked target、更好的 operator response target 都没有稳定超过 GRACE/matched GRACE。问题可能不是 target 还不够精细，而是 graph SSL 没有直接模拟下游“节点分类任务族”。

GRAFT 的 anchor：

> 与其预测某个 pretext target，不如让 encoder 在一族无标签生成的 plausible node-classification tasks 上学会可线性分离。

## Core Definition

给定真实图 `G=(A,X)`，构造无标签任务先验：

```text
tau ~ p_phi(tau | G)
y_i^tau = T_tau(A, X, i)
```

其中 `T_tau` 只能使用 features、topology、diffusion、role statistics 与预注册随机种子，不得读取真实 labels。

Encoder 训练目标：

```text
z_i = f_theta(A, X)_i
L_GRAFT = E_tau SupCon(z, y^tau) + lambda_bal L_task_balance + lambda_var L_collapse_guard
```

下游评估：

```text
freeze f_theta
train LogisticRegression on real train labels
select C on validation
report test once
```

## Task Generator

最小版 GRAFT 包含四类任务族：

- `feature_mixture`：随机 feature 子空间 + sign/quantile boundary + light graph smoothing。
- `topology_role`：degree、ego density、PageRank/diffusion、Laplacian PE bins 的 role buckets。
- `homophily_mrf`：从 seed logits 出发，用 Potts/Ising-like smoothing 产生不同 homophily 的 class assignments。
- `interaction`：feature residual 与 topology role 的组合边界。

所有 task family、class count range、temperature range、sampling count 必须写入 config，不能由 test set 或 downstream accuracy 决定。

## Non-Goals

- 不继续 ORBIT operator-response/basis target。
- 不做真实任务 pseudo-label self-training。
- 不做 kNN/PPR/CAST positive mining。
- 不做 AutoSSL pretext weight search。
- 不做 GraphPFN-style cross-graph foundation model。
- 不使用 validation/test performance 调 task generator。

## Expected Paper Contribution

1. 新问题定义：graph SSL 的 supervision object 应从 single pretext 转向 downstream task-prior fitting。
2. 新方法：same-graph label-free task generator + task-conditioned SupCon meta-risk。
3. 新控制实验：clustering SupCon、random task、feature-only、topology-only、PPR task、AutoSSL-style multi-pretext、task-shuffled、true-label SupCon upper bound。
4. 新诊断：synthetic-real train-label NMI/ARI、task diversity、task family contribution、linear readout transfer gap。

## Risks

- 可能被 reviewer 视为 pseudo-label SupCon。
- 任务生成器可能隐式调参空间过大。
- synthetic tasks 可能与真实 labels 低相关，导致 signal 弱。
- 若 GRAFT 只靠更多 epochs/tasks 获胜，会被判 unfair。
- 若 GRAFT 输给 clustering/PPR/AutoSSL controls，应立即 kill。
