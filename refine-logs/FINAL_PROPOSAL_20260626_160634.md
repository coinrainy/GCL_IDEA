## Motivation

CAST、BEACON、SPECTRA 分别排除了三个直觉路线：

- 低 overlap pair 不够；
- 高 label agreement / margin-rank pair gate 不够；
- rank / boundary-retention representation diagnostic 不够。

共同结论是：false-negative GCL 的问题不应再被定义为 pair correction 或 diagnostic regularization，而应被定义为 **SSL objective 与少标签 Logistic Regression 下游读出之间的错配**。

## Method Name

**PACER-GCL / Probe-Aligned Calibration for Evaluator-Ready Graph Contrastive Learning**

## Problem Anchor

在普通节点分类 GCL 中，最终评价是 frozen encoder + Logistic Regression。若预训练目标没有让少量 train labels 容易读出类别边界，即使 pair purity、overlap、rank 或 boundary diagnostics 更好，也可能不提升 accuracy。

PACER 的问题锚点：

> Can train-label logistic probes calibrate which anchors should receive instance-discrimination versus masked readout-preserving consistency, without using test labels or pseudo-positive mining?

## Method

### Stage 1: GRACE Warmup

训练一个短预算 GRACE warmup encoder，得到 `h_i`。

### Stage 2: Train-label Probe Calibration

只用 train labels 拟合 Logistic Regression probe：

```text
p(y | h_i) = softmax(W h_i + b)
```

定义 anchor readout fragility：

```text
q_i = instability_i + low_margin_i + augmentation_margin_drop_i
```

其中 margin 与 instability 均来自 train-label probe 或 train-split internal CV probe。Validation labels 只能用于选择预注册小集合超参，不参与训练目标或 test threshold。

### Stage 3: Objective Routing

PACER 不对 node pairs 打 same-class 分数。它只改变 anchor 的 objective type：

```text
if q_i is low:
    use standard GRACE instance discrimination
else:
    use masked logit-preserving consistency
```

对于 fragile anchors，两种 masked views 的 encoder output 必须保留 probe logit direction：

```text
L_logit = KL(stopgrad(probe(h_i_warmup)) || probe(h_i_view))
L_margin = max(0, margin_floor - margin_probe(h_i_view))
```

总目标：

```text
L = L_GRACE(stable anchors) + lambda_logit L_logit(fragile anchors)
    + lambda_margin L_margin(fragile anchors)
```

核心不是 loss reweighting，而是 objective type routing：fragile anchors 不再承受 full all-node instance discrimination。

## Non-goals

- 不做 kNN/PPR/CAST mining。
- 不做 cross-node positive mining。
- 不做 post-hoc relation smoothing。
- 不做 score fusion。
- 不继续 CAST C4/C5、BEACON gate、SPECTRA boundary/rank tuning。
- 不声称纯无监督；PACER 是 train-label-calibrated / semi-supervised GCL。
- 不用 test labels 做任何训练、阈值、probe、早停或超参选择。
- 不在 `project_dirty=true` 或缺少 exact run command 的状态下扩大到 pilot/formal。

## Novelty Risks

| Risk | Why serious | Required control |
|---|---|---|
| SupCon / semi-supervised GCL | 使用 train labels 会被归类到半监督 | supervised contrastive control; label-only probe control |
| Probe loss / linear evaluation overfit | 可能只是直接优化 evaluator | probe-only and train-only CV controls |
| Loss reweighting | q_i 可能被误解为 sample weight | scalar reweight control must not match full |
| Validation leakage | val 被过度用于目标设计 | train-only version and strict val-only selection log |
| Shuffled label artifact | labels may act as arbitrary regularizer | shuffled-label probe must fail |
| SupCon baseline | PACER 可能只是复杂半监督对比学习 | small-label supervised contrastive control must not match full |
| Provenance | dirty run 或缺少命令会使 pilot 不可审计 | result JSON must save exact command and dirty flag |
