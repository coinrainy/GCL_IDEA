# Research Proposal: SIVA-GCL-positive-core

## Problem Anchor

- **Bottom-line problem**：普通节点分类中的 GCL 过度依赖随机增强 positives 和真实节点 negatives；前者经 message passing 后可能预对齐、学习信号弱，后者在 hard-negative 区域容易引入 false negatives。
- **Must-solve bottleneck**：需要改变 positive training signal 的来源，而不是继续在 InfoNCE denominator / loss weight 上修补。
- **Non-goals**：不主打 loss trick；不把 virtual hard negative 作为主贡献；不使用 test label；不引入大 diffusion / LLM / KAN 模块；不把 smoke 写成性能 claim。
- **Constraints**：首轮基于 GRACE/GCA 代码，Cora seed=0 smoke，`1:1:8` split，frozen encoder + Logistic Regression evaluator。
- **Success condition**：SIVA-positive 生成的 view 比 GRACE/GCA augmentation 更远但语义稳定；post-GNN positive signal 更 informative；GraphMAE-only、critic-only、random intervention 不能解释其效果。

## Technical Gap

SPGCL 2026 指出 message passing 会使 positive pairs 在优化前已经相似，削弱 positive learning signal。已有 false-negative GCL 多从 negative side 修补：pair reweighting、hard negative mining、false-negative filtering、counterfactual hard negatives。用户明确指出 BOND 这类 loss-side 方法创新性不足。因此本方案转向 positive signal source：主动生成 **anti-prealignment positives**。

## Method Thesis

**One-sentence thesis**：SIVA-GCL searches for maximally different but semantically stable node-local intervention positives, using a masked-context semantic stability critic to make graph contrastive positives informative after message passing.

## Contribution Focus

- **Dominant contribution**：critic-constrained semantic-preserving intervention positive。
- **Optional supporting contribution**：positive informativeness diagnostics，包括 view distance、stability、post-GNN positive similarity、positive gradient norm。
- **Explicit non-contributions**：virtual-negative 不是主贡献；masked critic 不是单独方法；loss 保持 GRACE/GCA 风格。

## Proposed Method

### Complexity Budget

- Reused backbone：GRACE/GCA encoder、projector、optimizer、frozen LogReg evaluator。
- New component：轻量 masked-context semantic stability critic。
- New procedure：node-local intervention positive search。
- Excluded：diffusion generator、class-balanced negative graph generation、large learned augmentation policy、LLM/VLM teacher。

### System Overview

```text
Original graph + node features
        |
        v
Masked-context critic C learns stable ego semantic code q_i
        |
        v
For each anchor i, search intervention a over ego context:
  maximize distance(view_i(a), view_i(original))
  subject to stability C(view_i(a), q_i) >= threshold
        |
        v
Use selected SIVA-positive view in GRACE/GCA training
        |
        v
Evaluate frozen encoder with Logistic Regression
```

### Semantic Stability Critic

The critic `C` is trained with masked ego-context reconstruction:

```text
q_i = EMA(C_context(masked_ego_i))
L_critic = reconstruction_loss(masked_features, masked_edges/context)
```

It outputs a stability score:

```text
stab(i, a) = sim(q_i, q_i(a)) - recon_error(i, a)
```

The critic is used only to constrain intervention positives. A critic-only / GraphMAE-only baseline must be included to show it is not the whole method.

### Intervention Positive Search

For each anchor, candidate interventions include:

- feature dimension masking or replacement within ego context；
- edge dropout / edge keep around ego graph；
- neighbor feature masking；
- small ego-context substitution from low-risk neighbors。

Select:

```text
a_i^+ = argmax_a distance(view_i(a), view_i(original))
        subject to stab(i, a) >= gamma
```

Implementation smoke can start with greedy search over a small intervention pool, not a trained generator.

### Training Recipe

1. Train or warm up the semantic stability critic for a small fixed budget.
2. Generate SIVA-positive views every `K` epochs or precompute masks.
3. Train GRACE/GCA-style contrastive encoder using original view and SIVA-positive view.
4. Keep real-node negatives simple/random; do not introduce virtual negatives in main smoke.
5. Log positive informativeness metrics.

## Failure Modes and Diagnostics

- **Critic shortcut**：critic learns degree / identity rather than semantics。  
  Diagnostic：critic-shuffled and GraphMAE-only controls。

- **Random stronger augmentation explains gains**。  
  Diagnostic：random intervention positive matched for edit distance。

- **SIVA positive not actually anti-prealigned**。  
  Diagnostic：post-GNN positive similarity and positive gradient norm compared with GRACE/GCA。

- **Interventions destroy semantics**。  
  Diagnostic：stability curve, reconstruction error, LogReg drop。

## Novelty and Elegance Argument

SIVA-positive core is not another denominator correction. It changes the positive view generation contract: positives are selected to be maximally nontrivial while passing semantic stability. This differs from RGCL/GCA, which preserve salient rationale or adapt augmentation, and from SPGCL, which modifies propagation / positive sampling. It also avoids the CGC/BalanceGCL collision by not making virtual hard negatives the core.

## Claim-Driven Validation Sketch

### Claim 1：SIVA positives are more informative than ordinary graph augmentations

- Minimal experiment：Cora seed=0 smoke。
- Baselines：GRACE, GCA-style augmentation, random intervention positive。
- Metrics：view distance, post-GNN positive similarity, positive gradient norm, LogReg。
- Expected evidence：SIVA has higher nontrivial view distance while maintaining stability and useful positive gradients。

### Claim 2：The semantic critic is a constraint, not the entire source of gain

- Minimal experiment：GraphMAE-only / critic-only / critic-shuffled controls。
- Metrics：LogReg, stability, reconstruction, positive informativeness。
- Expected evidence：SIVA-positive beats critic-only and shuffled-critic controls。
