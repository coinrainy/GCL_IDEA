# Experiment Plan

**Problem**: Test whether node-wise contrastive eligibility can route anchors between full InfoNCE and positive-only consistency in Graph SSL.  
**Method Thesis**: Objective type switching for ineligible anchors must outperform same-score scalar weighting and known proxies, or the method is killed.  
**Date**: 2026-06-26

## Claim Map

| Claim | Why It Matters | Minimum Convincing Evidence | Linked Blocks |
|---|---|---|---|
| C1: Eligibility signal predicts alignment suitability beyond proxies | Without this, CER-GCL is just degree/uncertainty weighting | `q_i` retains residual predictive value after degree, uncertainty, drift-only, fs-only controls | B1, B3 |
| C2: Objective routing is not scalar weighting | This is the core technical delta | CER-GCL beats same-score scalar weighting on targeted bucket diagnostics, not only overall accuracy | B2, B3 |

## Fixed Pilot Definitions

- Split: project stratified random `1:1:8`, seeds 0-2 for pilot.
- Datasets: Cora and CiteSeer first.
- Evaluator: frozen encoder + Logistic Regression, selected with train/val only.
- Gate formula: `q_i = rank_norm(drift_i) + rank_norm(fs_residual_i)`, fixed `rho=0.8`.
- `s_loss`: diagnostic only, not a gate input.
- Denominator policy:
  - Default: ineligible nodes are not InfoNCE anchors, but remain stop-gradient denominator candidates.
  - Ablation: denominator-detach/mask for ineligible nodes.

## Experiment Blocks

### Block 0: Smoke And Collapse Check

- Claim tested: implementation does not collapse or corrupt evaluation.
- Dataset / split / task: Cora seed 0, `1:1:8`.
- Compared systems: vanilla GRACE, CER-GCL default.
- Metrics: embedding rank, per-dimension variance, route mask distribution, loss finite check, logreg_val smoke accuracy.
- Success criterion: no NaN, no collapse, route mask near fixed quantile, evaluator runs.
- Failure interpretation: stop before pilot.
- Priority: MUST-RUN.

### Block 1: Eligibility Proxy Isolation

- Claim tested: C1.
- Dataset / split / task: Cora/CiteSeer seeds 0-2.
- Compared systems: random gate, degree-only, uncertainty-only, drift-only, fs-residual-only, full `q_i`.
- Metrics: bucket-wise accuracy delta, post-hoc alignment benefit correlation, score correlations with degree/uncertainty.
- Success criterion: full `q_i` has residual signal beyond controls.
- Failure interpretation: kill eligibility claim or reduce to diagnostic-only.
- Priority: MUST-RUN.

### Block 2: Routing vs Weighting

- Claim tested: C2.
- Dataset / split / task: Cora/CiteSeer seeds 0-2.
- Compared systems: vanilla GRACE, global weak pressure, same-score scalar weighting, CER-GCL two-route objective, random routing.
- Metrics: frozen Logistic Regression, bucket-wise diagnostics, embedding rank/variance.
- Success criterion: CER-GCL improves targeted bucket diagnostics over scalar weighting.
- Failure interpretation: if scalar weighting matches, kill routing claim.
- Priority: MUST-RUN.

### Block 3: Denominator Policy Check

- Claim tested: negative-side gradients do not explain or destroy the intended intervention.
- Dataset / split / task: Cora seed 0-2.
- Compared systems: denominator keep-stopgrad, denominator detach, denominator mask.
- Metrics: route bucket alignment drift, accuracy, collapse checks.
- Success criterion: one policy is stable and does not create collapse; default is justified or replaced.
- Failure interpretation: revise objective before development.
- Priority: MUST-RUN.

## Stop / Go Rules

- `KILL`: scalar weighting matches CER-GCL and bucket diagnostics show no independent routing value.
- `KILL`: embedding collapse occurs and cannot be fixed without adding new modules.
- `PIVOT`: full `q_i` is only degree/uncertainty proxy.
- `REVISE`: denominator policy matters but one stable variant has signal.
- `GO`: C1 and C2 both pass on Cora/CiteSeer pilot; then proceed to 10-seed development.
