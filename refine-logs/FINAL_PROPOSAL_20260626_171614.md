## Method Name

**ORBIT-GCL / Operator-Response Basis Induction for Graph Contrastive Learning**

## Problem Anchor

Graph SSL for ordinary node classification has been over-focused on node-node agreement, false-negative correction, masked input reconstruction, or local diagnostic regularization. 本项目多轮 no-go 显示：更复杂的 pair mining、boundary/rank diagnostics、train-label routing、masked evidence group prediction 都没有稳定变成 frozen Logistic Regression accuracy。

ORBIT 的 anchor 是：

> 节点分类需要 class-readable perturbation sensitivity；自监督阶段应学习节点对不同 graph intervention operators 的响应函数，而不是继续学习节点相似性或输入重建。

## Method Thesis

ORBIT 固定一组无标签 graph operators，测量每个节点在这些 operators 下的 teacher response field，并训练 student encoder 从 clean / weak view 预测该 response field 的低秩 basis。目标是让表示显式携带节点对特征、结构、扩散、ego radius、位置扰动的敏感性模式。

## Core Definition

给定图 `G=(A,X)`、operator bank `Omega={O_1,...,O_M}`、teacher encoder `f_T`：

```text
h_i^0 = f_T(G)_i
h_i^m = f_T(O_m(G))_i
r_i^m = h_i^m - h_i^0
R_i = concat(r_i^1, ..., r_i^M)
b_i = LowRankProject(R_i)
```

Student encoder `f_S` 接收 clean / weak view，输出 `z_i=f_S(G_weak)_i`，prediction head `q` 预测 `b_i`：

```text
L_orbit = || normalize(q(z_i)) - stopgrad(normalize(b_i)) ||_2^2
```

最小 ORBIT full 只允许包含：

1. fixed operator bank
2. teacher response field
3. low-rank response-basis target
4. student basis prediction
5. held-out operator response diagnostic

Collapse guard、Barlow/VICReg、operator-code contrast、same-node consistency 不能作为主贡献；若后续使用，必须作为 explicit ablation。

## Operator Bank

初始 operator bank 只使用 label-free、可复现、低成本 operators：

- `feature_frequency_mask`：按 feature frequency / sparse feature bucket 做遮罩。
- `degree_preserving_edge_rewire`：保持 degree distribution 的局部边扰动。
- `diffusion_temperature_shift`：改变 diffusion / propagation temperature。
- `ego_radius_crop`：改变 ego receptive field radius。
- `positional_perturbation`：扰动 Laplacian / random-walk positional signal。

必须预注册 operator 参数，禁止用 test set 或 downstream accuracy 选择 operator。

## Differentiation

- vs GRACE：ORBIT 不用其他真实节点作为 negatives，也不优化 node-node denominator。
- vs GraphMAE / GraphMAE2：ORBIT 不重建 feature/edge/position，也不是 remask decoding；它预测 response basis。
- vs Graph-JEPA：ORBIT 不预测单个 target latent，而预测跨 operator 的 response function。
- vs TTER / D-SLA：ORBIT 不预测 transformation id 或 whole-graph discrepancy，而预测 node-level response basis。
- vs IRIS / WILLOW：ORBIT 不搜索 cross-node positives，不做 response-sibling mining。
- vs MEGA：ORBIT 不预测 discrete evidence group，而预测 continuous operator response field。

## Expected Paper Contribution

1. 新问题定义：operator response field for graph SSL。
2. 新方法：response-basis induction without labels or cross-node positives。
3. 新审计：held-out operator response generalization and shortcut controls。
4. 系统实验：与 GCL、masked SSL、JEPA、transformation/discrepancy prediction、response controls 的统一 protocol 对照。

## Risks

- 可能是 degree/feature/diffusion shortcut。
- 可能只是 EMA teacher / low-rank projection regularization。
- 可能与 TTER/D-SLA/Graph-JEPA 过近。
- Operator bank 选择可能引入隐式调参空间。
- Cora seed0 可能因为 operator target 难学而低于 GRACE。
