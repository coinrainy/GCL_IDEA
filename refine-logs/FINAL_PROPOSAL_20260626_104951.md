# Research Proposal: BOND-GCL

## Problem Anchor

- **Bottom-line problem**：普通节点分类场景下，图对比学习的 InfoNCE denominator 会把大量语义接近或同类节点当作独立负样本，造成 false-negative repulsion，损害 frozen encoder + Logistic Regression 下游分类。
- **Must-solve bottleneck**：不能只用 pair-level true-negative probability 或 hard-negative similarity；需要处理同一语义盆地被重复计入 denominator 后形成的 over-counted repulsive mass。
- **Non-goals**：不换 GNN backbone，不引入 KAN/LLM/大生成器，不使用 test label，不把 smoke/development 写成 SOTA。
- **Constraints**：基于当前 GRACE/GCA 风格实现；主协议 `1:1:8`；GCL evaluator 为 frozen encoder + Logistic Regression；首轮只允许 Cora seed=0 smoke。
- **Success condition**：BOND-cap-only 在 smoke 中数值稳定，false-negative mass exposure 下降，denominator effective number / uniformity 不崩，并优于 random-basin 与 pair-weight proxy；pilot 至少在 2/3 citation datasets 上不低于 GRACE。

## Technical Gap

已有 ProGCL/AUGCL/NodeSim 主要从 pair 角度修正 negatives；E2Neg 主要从 small effective negative sampling 角度减少 denominator；SPGCL 则从 positive pre-alignment 角度重新组织 positive signal。BOND-GCL 的切口是另一个 failure mode：在节点图中，一个 anchor 周围可能存在多个语义相近 basin，其中同一 basin 内的许多节点都被 denominator 重复当作负样本。即使每个 pair 的误伤不大，总 negative mass 也会持续推散类内结构。

## Method Thesis

**One-sentence thesis**：BOND-GCL replaces pair-wise denominator summation with basin-level negative mass aggregation so that each anchor-local semantic basin contributes bounded repulsive mass rather than an unbounded number of repeated false-negative forces.

## Contribution Focus

- **Dominant contribution**：basin-level negative mass aggregation operator for graph contrastive learning。
- **Optional supporting contribution**：offline false-negative mass exposure diagnostic，用真实标签仅在训练后诊断，不参与训练。
- **Explicit non-contributions**：不 claim 新 encoder；不 claim 生成 hard negatives；不 claim fully solved positive pre-alignment；不 claim formal/SOTA until formal protocol。

## Proposed Method

### Complexity Budget

- Frozen / reused backbone：GRACE/GCA encoder、augmentation、optimizer、frozen encoder evaluator。
- New trainable components：默认 0；BOND 是 loss-side operator。
- New non-trainable components：anchor-local basin assignment 和 denominator aggregation。
- Tempting additions intentionally excluded：多信号 learned basin scorer、LLM/VLM semantic teacher、KAN encoder、counterfactual generator。

### System Overview

```text
Graph views -> shared GNN encoder -> z1, z2
                         |
                         v
        anchor-local basin assignment B_i(j)
                         |
                         v
   pair logits s_ij = exp(sim(z_i, z_j) / tau)
                         |
                         v
 basin aggregation: M_i,b = sum_{j: B_i(j)=b} s_ij
                         |
                         v
 capped denominator: sum_b min(M_i,b, cap_i,b)
                         |
                         v
 optional overflow module (OFF by default in main claim)
```

### Core Mechanism

For anchor `i`, ordinary InfoNCE uses:

```text
D_i = exp(sim(z_i^1, z_i^2) / tau) + sum_{j != i} exp(sim(z_i^1, z_j^2) / tau)
```

BOND defines an anchor-local basin assignment `B_i(j) in {1...K_i}` and aggregates:

```text
M_i,b = sum_{j != i, B_i(j)=b} exp(sim(z_i^1, z_j^2) / tau)
D_i^bond = p_i + sum_b A_cap(M_i,b; c_i,b)
A_cap(M, c) = min(M, c)
```

The first implementation uses a fixed, non-learned basin constructor:

- primary constructor：PPR / adjacency-hop structural basin with optional feature-kNN tie-break；
- structure-independent control：random basin assignment with same size histogram。

Overflow replacement is not part of the main claim:

```text
D_i^bond+overflow = p_i + sum_b min(M_i,b, c_i,b) + lambda_overflow * R_i
```

where `R_i` is boundary representative mass. Main paper claim must first pass with `lambda_overflow = 0` or show overflow as clearly secondary.

### Training Plan

1. Warmup GRACE for `w` epochs, or compute initial basins from raw graph/feature only.
2. Train with BOND-cap-only loss.
3. Log denominator effective number, average cap activation ratio, basin size histogram, uniformity/alignment, and offline label-only false-negative mass exposure.
4. Select `cap_ratio` and simple discrete basin settings using train/val only.

## Failure Modes and Diagnostics

- **Failure mode**：BOND 等价于 pair reweighting。  
  Diagnostic：same pair weights / shuffled basins control。

- **Failure mode**：BOND 只是 E2Neg-style sampling。  
  Diagnostic：E2Neg-center control and denominator effective number comparison。

- **Failure mode**：cap 过强导致 representation collapse or poor uniformity。  
  Diagnostic：uniformity, embedding rank, denominator effective number。

- **Failure mode**：basin constructor 吃掉贡献。  
  Diagnostic：random basin same histogram, structure-independent basin, single-signal basin ablation。

## Novelty and Elegance Argument

BOND 的主张不是“哪个 pair 是假负”，也不是“少量 negatives 足够”。它把 false-negative damage 看作 distribution-level repeated repulsion：同一 anchor-local basin 中的多个节点以可加方式进入 denominator，导致语义近邻区域被过度推离。若反事实实验显示 shuffled basins / same pair weights 无法复现 BOND，而 BOND-cap-only 又优于 E2Neg-style center sampling，则该机制具有独立性。

## Claim-Driven Validation Sketch

### Claim 1：Basin-level repeated repulsion is a measurable failure mode

- Minimal experiment：Cora seed=0 smoke，训练后用 label-only offline diagnostic 计算同类 basin negative mass exposure。
- Baselines / ablations：GRACE、pair reweight proxy、random basin。
- Metric：false-negative mass exposure、cap activation ratio、class/basin exposure。
- Expected evidence：BOND 降低同类/近类重复排斥，而 random basin 不具备同等效果。

### Claim 2：Basin mass aggregation is not just pair reweighting or small negative sampling

- Minimal experiment：same pair weights / shuffled basins；E2Neg-style center negatives；BOND-cap-only。
- Metric：LogReg test@best、uniformity、denominator effective number、offline exposure。
- Expected evidence：BOND-cap-only 优于 shuffled/random basin 与 E2Neg-style center，且不靠 denominator collapse。
