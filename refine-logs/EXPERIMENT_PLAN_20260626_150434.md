# Experiment Plan: BEACON-GCL Smoke

## Status

- Stage: smoke planning
- Active idea: BEACON-GCL / Boundary-Eligibility Aware Contrastive Neutralization
- Trigger: CAST Certificate Pilot-A no-go
- Claim permission: none

## Main Question

Can a counterfactual boundary-utility gate filter false-negative candidate edits better than kNN/PPR/CAST/C4 mining and better than all-candidate neutralization?

## BE-M0: Cora Seed0 Sanity

| Field | Value |
|---|---|
| Dataset | Cora |
| Seed | 0 |
| Split | stratified random `1:1:8` |
| Encoder | GRACE frozen encoder |
| Evaluator | Logistic Regression with train/val C selection |
| Status | smoke |
| Timeout | < 2 GPU hours |

### Systems

| ID | System | Required |
|---|---|---|
| B0 | GRACE | yes |
| B1 | kNN smoothing | yes |
| B2 | CAST proxy smoothing | yes |
| B3 | C4 certificate smoothing | yes |
| B4 | all-candidate neutralization | yes |
| B5 | BEACON full gate | yes |
| B6 | shuffled utility gate | yes |
| B7 | no-boundary gate | yes |
| B8 | no-geometry gate | yes |

### Required Diagnostics

- accuracy: `test_at_best_val`
- accepted pairs per anchor
- pair label agreement, offline only
- overlap with kNN/PPR/CAST/C4
- utility mean/std for accepted and rejected pairs
- probe-margin delta on train/val
- representation rank delta
- local uniformity delta
- degree-bucket concentration

### Pass / Revise / Kill

- **GO to BE-M1** if B5 beats B1/B2/B3/B4 on Cora seed0 and has lower geometry damage than B4/B7.
- **REVISE** if B5 only beats GRACE but not kNN/CAST.
- **KILL** if B6 shuffled utility matches B5, or if B5 improvement comes from high overlap with kNN/CAST.
- **BLOCKED** if the runner cannot compute diagnostics without using test labels.

## BE-M2: Mechanism Diagnostics

Run only if BE-M1 has a non-negative signal.

- accepted vs rejected utility separation.
- boundary-risk ablation.
- geometry-risk ablation.
- threshold sensitivity chosen from train/val only.
- overlap partial correlation after controlling feature/embedding/PPR/degree.

## What Not To Run

- No Pilot-B.
- No paper table.
- No `/auto-review-loop`.
