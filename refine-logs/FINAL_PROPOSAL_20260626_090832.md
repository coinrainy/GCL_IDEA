# FINAL_PROPOSAL: IGT-GCL

标题：**Interval-Guarded Pair Targeting for Graph Contrastive Learning**  
简称：**IGT-GCL**  
任务：节点分类下的图对比学习  
状态：fresh restart 后的收缩版 proposal  
阶段决策：**REVISE**  
允许动作：method refinement、smoke/pilot planning  
禁止动作：直接 formal claim、SOTA/robust/comprehensive claim

## 1. Problem Anchor

节点级图对比学习常把同一节点的两种增强视图作为正样本，把 batch 或全图中其他节点作为负样本。这个设计与节点分类目标存在天然错位：非增强节点 `j` 对 anchor `i` 来说可能是同类、同语义、同社区的 candidate positive，也可能是语义不同但 embedding 相近的 true hard negative。

现有 false-negative 方法大多试图输出一个点式判断：

- `j` 是 false negative 的概率；
- `j` 是 true negative 的概率；
- `j` 与 `i` 的 node similarity；
- `j` 是否应该被扩为 positive。

但图节点 pair semantics 在训练早期和中期常常不可识别。点式 pseudo-label 容易过度自信：把不确定 pair 拉近，或把潜在同类 pair 推远。

## 2. Core Hypothesis

若 pair 的语义状态不可精确识别，则 contrastive objective 不应强迫每个 pair 接受一个点式 posterior。更合适的训练目标是一个 set-valued / interval-valued target：

```text
[l_ij, u_ij] = conservative lower bound and liberal upper bound
```

其中：

- 高 `l_ij` 表示多个独立信号一致支持 `i,j` 可能为同语义 pair，可以小权重吸引；
- 低 `u_ij` 表示几乎没有可靠信号支持 `i,j` 同语义，若当前 embedding 又相似，则可作为 reliable hard true negative；
- 低 `l_ij` 且高 `u_ij` 表示 ambiguous pair，训练时应 abstain 或弱化强排斥。

## 3. Method Summary

IGT-GCL 是 GRACE/GCA 风格方法的 loss-side 插件，不改变 encoder、augmentation family、data split 或 downstream evaluator。

### 3.1 Warmup and EMA Encoder

先用 vanilla GRACE/GCA 运行短 warmup，得到稳定但未充分收敛的 encoder，并维护 EMA embedding：

```text
z_i^ema <- m * z_i^ema + (1 - m) * z_i
```

warmup 的作用不是调参找最佳性能，而是收集 pair rank stability，避免完全依赖随机初始化阶段的相似度。

### 3.2 Pair Signals

对每个 anchor `i`，只在 top-k candidate pool 中构造 pair signals，避免全量 `O(N^2)` 存储：

| Signal | Meaning | Suggested implementation |
|---|---|---|
| `s_feat(i,j)` | feature-space semantic closeness | feature cosine rank or sparse top-k |
| `s_struct(i,j)` | graph proximity | PPR / shared-neighbor / k-hop rank |
| `s_embed(i,j)` | current semantic guess | EMA embedding cosine rank |
| `s_stab(i,j)` | temporal / augmentation reliability | rank stability across views/checkpoints |

所有 signal 归一化为 anchor-wise rank score in `[0,1]`，降低 dataset scale 影响。

### 3.3 Interval Construction

lower bound 使用保守一致性：

```text
l_ij = min(s_feat(i,j), s_struct(i,j), s_embed(i,j), s_stab(i,j))
```

或使用 trimmed conservative aggregator：

```text
l_ij = quantile_q_low({s_feat, s_struct, s_embed, s_stab})
```

upper bound 使用宽松并集：

```text
u_ij = max(s_feat(i,j), s_struct(i,j), s_embed(i,j), s_stab(i,j))
```

或使用 high quantile aggregator：

```text
u_ij = quantile_q_high({s_feat, s_struct, s_embed, s_stab})
```

这不是最终统计保证。pilot 的第一目标就是验证这些 interval 是否有可诊断意义：high-`l` precision、low-`u` true-negative precision、ambiguous false-negative enrichment 必须优于点式 posterior。

### 3.4 Pair Action Sets

对每个 anchor `i`：

```text
P_i = {j: l_ij >= tau_l}                         # conservative positives
H_i = {j: u_ij <= tau_u and sim(z_i,z_j) high}    # reliable hard true negatives
A_i = remaining candidate pairs with l_ij < tau_l and u_ij > tau_u
```

约束：

- `P_i` 有 per-anchor quota，避免大类/高 degree 节点吞掉少数类。
- `H_i` 必须同时满足 low upper bound 与 embedding-hard，避免只选 easy negatives。
- `A_i` 不作为普通 negative 强推远；可完全移除，或仅以 stop-gradient denominator 维持数值稳定。

### 3.5 Interval-Guarded InfoNCE

设 `p_i` 是 self-positive view，`sim` 是 cosine similarity，temperature 为 `tau`：

```text
num_i =
  exp(sim(z_i, z_p_i) / tau)
  + alpha * sum_{j in P_i} l_ij * exp(sim(z_i, stopgrad(z_j)) / tau)

den_i =
  num_i
  + sum_{j in H_i} beta_ij * exp(sim(z_i, z_j) / tau)
  + gamma * stopgrad(sum_{j in A_i} exp(sim(z_i, z_j) / tau))

L_i = -log(num_i / den_i)
```

其中：

```text
beta_ij = (1 - u_ij) * hard_rank_weight(i,j)
```

默认 pilot 可令 `gamma = 0` 或一个很小常数。若 `gamma > 0`，ambiguous denominator 必须 stop-gradient，避免隐式 repulsion。

## 4. Claimed Contribution Boundary

IGT-GCL 只能主张以下较窄贡献：

1. **Set-valued pair target construction**：在图节点 GCL 中用 lower/upper-bound pair target 表达 false-negative / hard-negative ambiguity。
2. **Three-way target action**：lower-bound attraction、upper-bound reliable hard negative、ambiguous abstention 共同定义 contrastive pair 动作。
3. **Diagnostic-first design**：方法自带 interval coverage / precision / enrichment 诊断，允许判断它是否真的优于点式 posterior。

不能主张：

- 首个 false-negative GCL 方法；
- 首个 PU-GCL 方法；
- 普遍解决 hard negative / false negative 混淆；
- SOTA、robust、comprehensive；
- interval 一定是核心，除非 full interval 显著优于点式 posterior 和简单变体。

## 5. Novelty Positioning

相对 ProGCL：IGT-GCL 不只估计 true-negative probability，而是用 lower/upper bounds 分别控制 positive attraction、hard negative repulsion 和 ambiguous abstention。

相对 Node Similarity / IFL-GCL：IGT-GCL 不把 pair semantics 压成一个 similarity/posterior 分布，而保留 epistemic uncertainty interval，并显式测试 midpoint posterior 是否足够。

相对 affinity uncertainty HNM：IGT-GCL 不是给所有 negatives 连续 uncertainty weight，而是用 interval 驱动三种互斥动作。

相对 NML-GCL：IGT-GCL 不引入 learnable negative metric network 或 bi-level training；它更轻量，但必须靠诊断和公平 ablation 证明不是弱化版。

## 6. Minimum Viable Implementation Scope

后续若用户批准实现，最小改动应只落在 GRACE/GCA loss 侧：

- 增加 candidate pair top-k 缓存；
- 增加 feature/structure/EMA/stability signal 计算；
- 增加 interval pair masks；
- 增加 interval-guarded loss；
- 增加诊断日志：pair precision/enrichment、mask sizes、gradient mass、alignment/uniformity。

不应在第一版加入：

- 新 encoder；
- KAN / large metric learner；
- prototype clustering；
- label-supervised pair calibration；
- 复杂 heterophily-specific routing。

## 7. Failure Modes

| Failure | Interpretation | Decision |
|---|---|---|
| full interval ≈ midpoint posterior | interval 不是核心 | pivot |
| full interval ≈ only negative downweight | 收益来自 denominator shrinkage | pivot to gradient budget |
| full interval ≈ only positive expansion | 收益来自 positive scarcity 修正 | pivot to conservative NodeSim |
| ambiguous set 无 false-negative enrichment | abstention 没有语义依据 | kill or diagnostic-only |
| high-`l` precision 低 | positive expansion 不可靠 | revise interval construction |
| low-`u` hard negatives 不比普通 hard negatives 更纯 | hard-negative claim 不成立 | revise or kill |
| Amazon/heterophily 大幅下降 | 机制依赖强同配 | restrict claim or pivot |

## 8. Reviewer-Informed Revision

fresh review verdict 是 **REVISE**, score **5.5/10**。本 proposal 已按 reviewer 要求做三点收缩：

1. 名称从 generic `IPU-GCL` 改为 **IGT-GCL / Interval-Guarded Pair Targeting**，避免与已有 PU-GCL 线混淆。
2. 主叙事从“新的 PU framework”收缩为“避免点式 pseudo-label 过度自信的 interval target construction”。
3. 把 midpoint posterior、NodeSim/IFL-style point target、ProGCL-style weighting、only-downweight、only-expansion、matched random interval 写成强制 ablation。

## 9. Decision

**Decision: REVISE.**

IGT-GCL 值得进入下一轮 smoke/pilot planning，但当前证据只到 idea/refinement 层级。必须先完成最小实现与 pilot 诊断，且在 full interval 未击败点式 posterior 和简单替代解释前，不得进入 formal claim。
