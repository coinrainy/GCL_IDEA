## Method Name

**MEGA-GCL / Masked Evidence Group Assignment for Graph Contrastive Learning**

## Problem Anchor

False-negative GCL 反复失败，不是因为还没有找到足够好的假负 pair，而是因为 node-node contrastive denominator 对普通少标签节点分类来说可能是错误任务。修正 pair、rank、probe margin 都在改善代理变量，但没有训练表示保留 class-readable ego evidence。

## Method Thesis

MEGA 将 contrastive target 从真实节点替换为 masked ego evidence groups。它不问“哪些节点应当靠近”，而问“给定 anchor 的剩余 ego context，被遮盖的属性/结构证据属于哪个离散 evidence group”。这样保留 discriminative pretext，又绕开 node-node false negatives。

## Evidence Groups

MEGA 使用无标签、预注册 evidence tokenizer：

1. Feature evidence：对 node features 做 feature dropout sensitivity / sparse feature hash / PCA sign bucket，得到 feature group id。
2. Structural evidence：degree bucket、ego edge density bucket、two-hop role bucket。
3. Residual evidence：feature 与 1-hop smoothed feature 的 residual norm bucket。

这些 group 不使用 train/val/test labels，也不使用 kNN/PPR/CAST relation。

## Objective

对每个节点生成两种 masked ego views：

- context view：保留大部分 ego evidence；
- target-masked view：mask 掉一个 evidence group。

Encoder 输出 `h_i`，projector 预测 masked evidence group：

```text
L_evidence = CE(g_phi(h_i_context), group_i_masked)
L_view = consistency(h_i_context, h_i_target_masked)
L = L_evidence + lambda_view L_view + lambda_uniform L_uniform_guard
```

`L_uniform_guard` 只作为 collapse guard，不是 SPECTRA-style rank/boundary diagnostic regularizer。

## Non-Goals

- 不做 kNN/PPR/CAST mining。
- 不做 cross-node positive mining。
- 不做 pair score fusion。
- 不做 post-hoc relation smoothing。
- 不继续 CAST C4/C5、BEACON gate、SPECTRA rank/boundary、PACER probe routing。
- 不声称已验证性能；当前只是 proposal。

## Key Controls

- GraphMAE-like reconstruction：证明 MEGA 不是普通 masked autoencoder。
- GraphMAE2-lite latent/remask prediction：证明 MEGA 不是 latent masked prediction。
- GCMAE-style masked + contrastive hybrid：证明 MEGA 不是 MAE+CL stacking。
- VQ/codebook autoencoder：证明 MEGA 不是 graph VQ-GAE。
- SwAV/prototype assignment：证明 MEGA 不是 prototype clustering。
- MINT token-corruption contrast：证明 MEGA 不只是 token corruption contrast。
- Random evidence groups：证明 evidence tokenizer 有信息。
- Shuffled evidence targets：证明不是 regularization artifact。
- Feature-only / structure-only / degree-only：定位有效证据来源，并排除 BoW feature shortcut 或 degree/community shortcut。
- No-mask contrastive：证明 mask 任务必要。
- Matched-parameter GRACE：控制 capacity。
